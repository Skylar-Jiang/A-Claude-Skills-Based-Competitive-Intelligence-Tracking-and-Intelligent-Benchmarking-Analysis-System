## 采集与异常处理

`modules/data_loader.py` 的 `fetch_agri_daily_reports` 对每页最多尝试 3 次，请求间隔 2 秒。单页失败写入 `fetch_errors`，其他成功页仍可保留；若全部失败则抛出明确异常。页面缺少正文或发布时间时解析失败，不生成伪记录。

`modules/source_manager.py` 把直接解析结果保存到 raw，再做关键词过滤、公开 URL 校验和去重后合并到 processed。去重同时检查 `record_id` 和规范化链接。

2026-07-11 真实运行结果：

```text
第一次：raw_count=8, count=8, duplicate_count=0, fetch_errors=[]
第二次：raw_count=8, count=0, duplicate_count=8, fetch_errors=[]
```

## Sample 与真实数据

旧 `data/raw/*_manual.csv` 使用 `manual.local` 链接，只是课堂手工样例。`/ingest/csv` 检测到非公开链接或缺少来源/发布时间后，会写入 `data/samples/imported_records.csv`，返回 `mode=sample, indexed=false`，不会覆盖 processed 或触发正式 RAG。`/ingest/manual` 同样只写 sample。

## 已知问题与扩展

- 当前只有一个官方发布单位的多个时间点，能验证链路，不能做区域供应源横向价格对比。
- 固定 URL 适合可复现验收；扩展时应增加官方列表页增量发现和时间水位。
- 页面可能限速或变更 HTML；Parser fixture 位于 `tests/fixtures/agri_daily.html`。
- 新网页、RSS、政府公告、API/CSV 的接入位置和升级条件见 `docs/当前阶段边界与后续扩展.md`。

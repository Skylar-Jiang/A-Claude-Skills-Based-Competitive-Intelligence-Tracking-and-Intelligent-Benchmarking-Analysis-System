# 数据与接口约定

## 标准记录

每条正式记录包含：`record_id`、`title`、`content`、`source_name`、`source_url`、`source_type`、`published_at`、`collected_at`、`competitor`、`dimension`。`record_id` 由规范化 URL 和内容生成；同一链接重复采集不新增记录。

真实记录的 `source_url` 必须是可追溯的 HTTP(S) 公共页面，禁止 `.local`、`example.*` 或空链接。Mock/sample 记录必须显式标记，且不进入正式 processed/RAG。

## 存储

- raw：`data/raw/real/{source_id}.csv`，保存解析器直接产生的字段。
- processed：`data/processed/intelligence_records.csv`，保存清洗、过滤和去重结果。
- RAG：`data/processed/rag_index.json` + `chroma_db/`，metadata 保留来源与发布时间。
- 日志：`logs/collection_jobs.jsonl`、`logs/skill_trace.jsonl`。

## 模式

- `mode=real`：调用真实网络/模型；失败返回明确错误，绝不回落 Mock。
- `mode=mock`：只用于无 Key 的格式和 UI 联调；响应显式包含 `mode=mock`。
- 没有证据：不调用模型，返回 `insufficient_evidence=true`、空结论和可执行提示。

## API

正式前缀不做整体重命名，新增 `GET /records` 和 `POST /analysis/run` 补齐查询与统一分析。成功响应保留各领域模型；错误统一为：

```json
{"success": false, "error": {"code": "stable_code", "message": "human readable message"}}
```

兼容接口保留 `/analyze`、`/analyze/multi-agent`、`/skills/{skill_name}/run`，但文档标明其迁移目标。

## 失败与修复记录

1. 首次真实批量采集在第二页遇到 HTTP 403，原实现使整批失败。增加浏览器标识、2 秒间隔和逐页错误记录后，8/8 成功。
2. 一次真实模型分析因 1200 token 上限截断 JSON，API 正确返回 502。将正式分析上限提高到 2200 并增加回归断言后，重试返回 200。
3. 旧 `tests_local.py` 仍假设虚拟手工 CSV 进入正式 RAG，边界调整后失败。测试改为验证手工 CSV `indexed=false` 及真实官方证据，随后通过。
4. `python -m py_compile modules\*.py` 在 PowerShell 不展开通配符，改用 `compileall` 后通过；这是命令问题，不是代码失败。
5. 代码审查发现 `force=true` 越过 disabled、运行期 JSON 写后不读、无证据先初始化模型等问题；补充 11 项回归覆盖后修复，`raw/real` 清理为单一官方文件。

## 未解决问题

- FastAPI TestClient 输出 Starlette/httpx 弃用警告；当前测试通过，后续升级到兼容版本组合。
- Hugging Face 未配置 token 时有速率警告；模型已缓存且本次成功，部署环境建议配置镜像/缓存或 token。
- 真实模型输出仍具有外部服务不稳定性，系统会返回 502/503，不保证每次第三方调用成功。
- 当前真实数据只有一个官方来源的多个时间点，不足以形成区域供应源成交价横向对比。

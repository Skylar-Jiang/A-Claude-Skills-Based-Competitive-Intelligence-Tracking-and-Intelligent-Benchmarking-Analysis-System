# 当前阶段边界与后续扩展

## 当前阶段必须完成

- 业务对象：公开农产品批发市场日度动态，重点验证黄瓜及蔬菜价格/涨跌信息；不把手工区域样例当作真实区域报价。
- 数据来源：中国农业农村信息网“农产品批发价格 200 指数”日度公开报告，一批多个发布日期页面。
- 最小数据范围：至少 5 条可访问、不同发布时间的真实公开记录，保留来源名称、原始链接、发布时间、采集时间和内容摘要。
- 主链路：专用解析 -> 原始记录保存 -> 清洗去重 -> processed 保存 -> RAG 重建/检索 -> LangChain 证据分析 -> FastAPI -> 报告。
- 核心接口：健康、数据源列表、手动采集、记录查询、RAG 更新、检索、正式/Mock 分析、报告生成与历史报告。
- 测试：解析、字段约定、去重幂等、RAG 来源返回、证据不足、Mock/真实分离、API 错误与完整链路。

## 当前阶段暂不完成

- 大规模全网采集、分钟级价格、非公开交易/库存/物流数据。
- 扩展到多个行业、复杂多市场标准化价格比较。
- 绕过登录、验证码、付费、访问控制或严格反爬。
- 前端、复杂权限、异步任务队列、分布式并发、云集群、Docker、LangSmith。
- 为了形式引入复杂 Multi-Agent；保留现有入口，但以稳定证据 Chain 为正式主链。

## 扩展路线与接入位置

### 数据源

- 网页源：在 `modules/data_loader.py` 增加专用 Parser，并在 `modules/source_manager.py` 注册 source type；配置写入 `config/sources.yaml`。
- RSS：复用 `fetch_rss`，补充来源字段映射和 Parser fixture 测试。
- 政府公告：为列表页和详情页分别实现解析器，保留公告编号、发布机关和发布时间。
- API/CSV：实现 `MultiSourceDocumentLoader` 对应 loader；CSV 必须包含真实 URL 或明确标记为 sample，不能进入正式库。
- 启停与测试：使用 `enabled` 配置；每个 Parser 在 `tests/fixtures/` 保存最小公开结构 fixture，并增加解析测试。

### 业务对象和分析维度

- 增加竞品、品类、地区和市场：先扩展 `config/sources.yaml` 关键词/对象；页面结构不同则新增 Parser。
- 新维度：在 `modules/prompts.py` 增加证据约束 Prompt，在 `modules/agent_core.py` 注册 Agent，必要时在 `modules/skill_core.py` 暴露兼容 Skill。
- 需要新工具调用时扩展 `modules/tools.py`；需要不同流程编排时扩展正式 LangChain 分析链，而非复制 API 逻辑。

### 数据规模

- 单源超过 1 万条或需要复杂查询时，将 CSV/JSONL 迁移 PostgreSQL；`modules/data_loader.py` 保持记录契约，替换持久化实现。
- 页面增多时增加游标、时间水位、失败重试队列和归档；调度入口位于 `modules/scheduler.py`。
- Chroma 数据量显著增加时按业务对象/日期分 collection，并在 `modules/rag_chain.py` 增加增量 upsert、归档和重建策略。
- 并发前先增加缓存命中率、采集限速和基准测试，避免对公开源造成压力。

### 分析能力

- 趋势与横向对比：在真实数据覆盖多个日期/市场后增加确定性统计 Tool，再把统计结果作为证据交给 Chain。
- 风险、机会、日报/周报/月报：扩展 `modules/prompts.py` 和 `modules/report_writer.py`，保持证据 ID/URL。
- 多 Agent：当不同维度确有独立数据与评估集时，再用 LangGraph 并行节点替代当前顺序 Orchestrator。
- 模型分层、评估和反馈：在 `modules/llm_client.py` 配置角色模型，在 `tests/` 增加固定证据评测集；后续接 LangSmith。

### 系统工程

- 前端看板和图表：后续消费 FastAPI 的稳定模型，不在当前阶段开发。
- 用户配置、权限和鉴权：从 `verify_api_key` 迁移到用户/角色模型。
- Docker、云部署、监控：增加容器健康检查、持久卷、结构化日志和指标；入口仍为 `main.py`。

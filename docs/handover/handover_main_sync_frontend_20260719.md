- 最新上游基线：`origin/main` 的 `a772917 complete peer-group marketing integration`

## 上游新增能力

本轮 `main` 主要补齐了以下后端能力：

1. 运营决策 Agent 输出完整的上市营销策略，包括营销目标、目标客群、价值主张、定价、渠道、传播和上市动作。
2. 报告新增 `launch_marketing_strategy` 章节。
3. 报告证据索引新增面向用户的 `证据N` 标签、标题、类型名称、支持摘要和详情链接，机器证据 ID 退居审计详情。
4. 新增单条证据详情接口：`GET /api/v1/analysis-runs/{run_id}/evidence/{evidence_id}`。
5. 人工复核范围拆分为“证据审校复核”和“报关归类复核”，避免报关确认需求被错误解释为审校失败。
6. 新增离线商品类型盘点脚本和类型库存；该能力用于数据治理，不是在线品类推荐接口，因此前端仍保留可搜索、可选择、可自定义的品类输入。

## 前端适配内容

### 1. 新增“营销策略”独立页面

- 新增侧栏入口“营销策略 / 定位与上市动作”。
- 展示营销目标和产品定位。
- 以卡片形式展示目标客群、价值主张、定价策略、渠道策略、传播策略和上市动作。
- 任务尚未生成报告时展示真实空态，不伪造营销结论。

修改原因：新版后端已经把运营决策从简单行动清单升级为完整上市策略，继续挤在 Agent 详情或 Markdown 报告中不利于阅读和执行。

### 2. 新增“证据中心”独立页面

- 新增侧栏入口“证据中心 / 来源与原文详情”。
- 对接整批证据接口和新版单条证据详情接口。
- 列表优先展示 `证据N`、友好标题、证据类型和支持摘要。
- 详情区展示来源、原文摘要、发布时间、数据来源和审计元数据。
- 报告 Markdown 中的证据链接会在前端证据中心打开，不再把接口 JSON 直接展示给用户。

修改原因：UUID、ASIN 等机器标识不适合作为主要阅读入口；友好索引能够让普通用户理解引用关系，同时保留审计追踪能力。

### 3. 拆分人工复核边界

- 审校页新增“证据审校复核”和“报关归类复核”两张状态卡。
- 分别读取 `evidence_audit_manual_review_required` 与 `customs_broker_review_required`。
- 兼容旧报告中的 `manual_review_required`。

修改原因：证据事实审校与正式报关归类属于不同责任边界，必须在 UI 中分开表达，避免把关税候选项需要报关行确认误判成四号 Agent 审校失败。

### 4. 流程与导航更新

- Agent 流程图在运营决策后增加“营销策略”输出节点，再进入报告生成。
- 侧栏支持 7 个功能入口并保持独立滚动。
- 新页面补充桌面端、平板端和移动端响应式样式。
- 延续深色科研控制台、液态玻璃、粒子背景和减少动态效果的无障碍设置。

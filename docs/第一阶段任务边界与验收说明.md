# 第一阶段任务边界与验收说明

## 1. 阶段边界

依据《竞品情报工程规范》和《项目开发周期表》，第一阶段覆盖第 1-5 天任务：

1. 开发环境搭建。
2. LLM 与 LangChain 基础环境配置。
3. 竞品监控维度分析。
4. 舆情/价格数据采集样例准备。
5. RAG 向量知识库搭建与基础检索验证。

第一阶段不包含多 Agent 深度协作、Skill 化完整封装、FastAPI 全接口、定时报告、LangSmith、Docker、云端部署和前端看板。这些属于后续阶段。

## 2. 当前完成判断

当前仓库已经完成第一阶段的大部分功能性工作：

- 环境文件已具备：`requirements.txt`、`.env.example`、`setup.py`。
- 数据目录已具备：`data/raw/`、`data/processed/`、`data/reports/`。
- 数据源配置已具备：`config/sources.yaml`。
- 手工 CSV 样例已具备：盒马、叮咚买菜、美团买菜、京东生鲜、促销、行业新闻、社交舆情、投诉舆情。
- 网页/RSS/CSV 加载能力已具备：`modules/data_loader.py`。
- 基础清洗与过滤能力已具备：`modules/filtering.py`、`modules/semantic_filter.py`。
- Chroma RAG 索引能力已具备：`modules/rag_chain.py`。
- RAG 查询工具已具备：`modules/tools.py`。

当前已在项目虚拟环境中补齐第一阶段依赖并通过 `tests_local.py` 本地验证。后续维护时仍需保持数据维度命名一致，避免舆情数据进入知识库后无法被 `sentiment` 维度检索。

## 3. 第一阶段验收标准

第一阶段完成应满足以下可验证条件：

1. 能创建或使用项目虚拟环境，并成功安装 `requirements.txt`。
2. 能读取 `config/sources.yaml` 中的数据源配置。
3. 能导入至少一个手工 CSV 数据源。
4. 能生成或重建 `data/processed/rag_index.json`。
5. 能通过 RAG 查询返回带 `source_url` 的证据片段。
6. 能保留来源、采集时间、竞品、维度等元数据。
7. 不绕过登录、验证码、反爬，不抓取隐私、付费或未授权内容。

推荐本地验收命令：

```powershell
.\venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python tests_local.py
```

如果只验收第一阶段，可重点观察 `tests_local.py` 中的 `test_data_and_rag`、`test_collection_and_dashboard`。

## 4. 风险和边界说明

公开网页采集只能保证“可获取公开页面文本”，不能保证稳定拿到平台 App 内的实时价格、库存、配送时间和用户评论。知乎、小红书、微博、黑猫投诉、App 评论等来源应优先采用公开样例 CSV、授权 API 或人工导入方式。

因此，第一阶段的合理目标是完成“可追溯样例数据 + 可重建向量库 + 可检索证据片段”，不是完成细粒度实时爬虫。

## 5. 下一阶段衔接

第一阶段闭合后，第二阶段再继续补：

1. RAG 问答链和结构化情报提取。
2. 价格、新品、舆情 Agent 分工。
3. Skill 封装与统一输入输出。
4. FastAPI 接口和 Swagger 演示。
5. 报告生成、定时任务、部署和链路追踪。

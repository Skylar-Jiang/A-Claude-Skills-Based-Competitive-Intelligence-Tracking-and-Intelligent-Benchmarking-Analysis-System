## Mock、占位或未进入主流程的功能

- `data/raw/*_manual.csv` 共 28 条为手工样例，来源为 `手工公开样例`，链接使用不可访问的 `manual.local`，不能作为真实公开数据。
- `conversation_crawler.py` 默认允许网页失败后使用内置 HTML，并把状态写成 `success_with_local_sample`，存在失败伪装成功风险。
- `competitor_analysis_chains.py` 使用 LangChain `RunnableSequence`，但只是命令行演示，未进入正式 FastAPI 分析流程。
- `provider=mock` 是部分 Skill 接口默认值；真实 provider 失败时 `skill_core.run_agent_or_mock` 会回落 Mock，违背真实/Mock 分离要求。
- 当前测试大量使用测试 embedding、Fake LLM、`example.test` 和 `example.local`，能验证局部逻辑，不能证明真实网络、真实数据或真实模型链路。
- 报告目录中已有多批输出，但其证据来自手工样例或 Mock，不能作为本次真实链路验收结果。

## 当前主要链路

`config/sources.yaml -> source_manager -> data_loader -> filtering/semantic_filter -> data/raw/auto_collected_records.csv -> rag_chain/Chroma -> tools -> Agent/Skill -> report_writer -> FastAPI`

该链路结构存在，但原始/清洗数据未真正分层，正式 Agent 直接调用自研 OpenAI-compatible 客户端，LangChain Chain 未参与正式分析。

## 数据真实性与追溯

- 配置了农业农村部价格系统和商务部页面，但配置只有入口首页，通用正文抓取无法稳定生成一批结构化业务记录。
- 现有 28 条手工样例含虚拟链接，`auto_collected_records.csv` 也是这些样例的派生结果。
- `IntelligenceRecord` 缺少独立的 `source_name` 和 `published_at`；发布时间被混入 `collected_at`，无法区分发布与采集时间。
- `load_project_records` 同时扫描 raw 和 processed 下全部 CSV，会把示例数据直接带入 RAG，真实与示例知识库没有边界。

## 重复逻辑与接口问题

- `Agent` 与 `Skill` 各自包含相似的检索、分析、汇总和报告逻辑；兼容入口可以保留，但应指定一条正式链路。
- `/analyze`、`/analyze/multi-agent`、`/skills/{name}/run` 输入字段和 provider 语义不同。
- API 错误大多是字符串 `detail`，缺少稳定错误码；真实网络和模型失败的 HTTP 状态也不统一。
- `load_sources()` 总是优先读取 YAML，而新增/删除数据源写入 JSON，导致运行期数据源变更下次读取无效。
- README 和接口说明引用不存在的 `data/raw/hema_price_manual.csv`，配置、代码和文档不一致。

## 优先级

### P0

1. 虚拟链接手工样例进入正式 RAG，无法满足真实数据要求。
2. 采集失败可回落本地样例并显示成功；真实 Skill 失败可回落 Mock。
3. 原始与清洗数据未区分，缺少来源名称和发布时间字段。
4. 正式分析没有实际使用 LangChain/LangGraph Chain。
5. YAML/JSON 双配置写读不一致，数据源管理接口修改不能生效。

### P1

1. API 请求、响应和错误结构不统一，分析入口重复且语义不同。
2. RAG 会扫描所有 CSV，不能保证正式知识库只由真实记录构建。
3. 测试不是标准测试目录，运行慢，真实链路无集成覆盖。
4. 文档含不存在文件、过期业务描述和超前完成声明。

### P2

1. Agent、Skill 与独立 Chain 的重复逻辑可在后续版本逐步收敛。
2. 调度器、Memory 和报告历史目前使用本地文件，数据量增大后需迁移数据库。
3. `requirements.txt` 是完整环境冻结，后续可拆分直接依赖与锁文件。

## 本次修复结论

上述 P0 已完成针对性处理：正式 RAG 只读可追溯 processed；真实采集不再回落本地样例；原始与清洗数据分层并强制来源/发布时间；正式分析进入 LangChain RunnableSequence；运行期数据源 JSON 可持久重载。代码审查后又修复了强制采集越过 disabled、手工源污染 `raw/real`、无证据先初始化模型和 RAG 缓存未刷新问题。P2 保留为后续工作。

# 项目交接文档

## 当前完成状态

后端主链已经用 8 条农业农村部真实公开记录验证。真实采集、raw/processed、RAG、LangChain 分析、FastAPI、真实模型调用和报告保存均有实际运行记录。前端、大规模数据、生产部署不在当前范围。

## 目录与入口

- `main.py`、`modules/api_server.py`：服务入口和 HTTP 契约。
- `config/sources.yaml`：静态数据源默认配置；API 修改后由忽略版本控制的 `data/config/data_sources.runtime.json` 保存运行状态。
- `modules/data_loader.py`：记录契约、公开 URL 校验、专用 Parser、读写。
- `modules/source_manager.py`：采集、分层保存、去重、日志。
- `modules/rag_chain.py`：分块、HF embedding、Chroma、检索。
- `modules/analysis_chain.py`：正式 LangChain RunnableSequence 和模式边界。
- `modules/agent_core.py`、`modules/skill_core.py`：兼容 Agent/Skill 调度层。
- `modules/report_writer.py`：JSON/Markdown 报告。
- `tests/`：新的单元/集成测试；`tests_local.py`：原有回归。

## 安装与启动

按 README 创建 venv、安装 `requirements.txt`、复制 `.env.example`。真实模型必须配置有效 OpenAI-compatible Key、网关和模型名。启动：

```powershell
$env:COLLECTION_SCHEDULER_ENABLED="false"
.\venv\Scripts\python.exe -m uvicorn modules.api_server:app --host 127.0.0.1 --port 8000
```

## 操作流程

1. 真实采集：`POST /collect/run?force=true&use_llm_filter=false`。
2. 查看数据：`GET /records`；raw/processed 文件见数据源文档。
3. 更新 RAG：`POST /rag/rebuild`。
4. 检索：`GET /rag/search?query=黄瓜价格涨跌`。
5. 分析：`POST /analysis/run`，正式用 `mode=real`。
6. 报告：兼容接口 `POST /report/generate`，或将正式分析结果交给 `save_report`；历史用 `GET /reports`。

## 真实与示例边界

`data/raw/real/` 和 `data/processed/intelligence_records.csv` 是正式数据。旧 `*_manual.csv`、`manual.local` 和显式 Mock 只用于格式/测试；非公开 CSV 被写入 `data/samples/` 且 `indexed=false`。

## 当前限制

- 只有农业农村部日度报告，不能声称覆盖区域成交价、库存或供应合同。
- 价格分析是对公开日度涨跌信号的证据总结，不是采购建议或预测模型。
- Chroma、CSV、JSONL 适合当前小规模；扩大数据量前需数据库和增量向量策略。
- 第三方模型/网络可能失败，API 会明确返回错误。

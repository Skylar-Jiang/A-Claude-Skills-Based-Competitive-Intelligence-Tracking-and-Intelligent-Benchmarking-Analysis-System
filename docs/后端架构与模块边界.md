# 后端架构与模块边界

正式链路为：`公开源 -> 专用 Loader/Parser -> raw 快照 -> 清洗与去重 -> processed 标准记录 -> RAG -> LangChain 证据分析 -> FastAPI/报告`。

- `config/sources.yaml`：版本化数据源与业务范围的静态默认配置；首次 API 新增/修改/删除后，`data/config/data_sources.runtime.json` 成为运行期权威存储（该文件被忽略，不作为版本化数据）。
- `modules/data_loader.py`：标准记录、专用解析、清洗和文件读写。
- `modules/source_manager.py`：采集编排、幂等合并、日志；不承担页面解析。
- `modules/rag_chain.py`：分块、向量持久化、关键词降级和带来源检索。
- `modules/analysis_chain.py`：正式 LangChain 证据分析、结构校验和证据不足短路。
- `modules/agent_core.py`：兼容 Agent 注册与维度调度，调用正式分析链。
- `modules/skill_core.py`：兼容 Skill 接口；不得在真实失败时回落 Mock。
- `modules/api_server.py`：HTTP 模型、路由、错误映射，不复制业务逻辑。
- `data/raw/real/`：采集后未经过业务过滤的原始提取记录。
- `data/processed/intelligence_records.csv`：正式清洗记录，是 RAG 的数据入口。
- `data/samples/`：演示/Mock 样例；不会进入正式 RAG。

当前不删除旧入口。`POST /analysis/run` 作为正式统一分析入口；`/analyze` 与 Skill/Agent 接口保留兼容并在文档标注。
